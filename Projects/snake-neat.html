Snake NEAT

NeuroEvolution of Augmenting Topologies applied to Snake Game

C++ SFML NEAT Algorithm Neural Networks Genetic Algorithm

About the Project

Teaching an AI to play Snake — not by programming the rules, but by evolving neural networks until they figure it out

The Idea

Snake NEAT is my implementation of NeuroEvolution of Augmenting Topologies (NEAT). Instead of training a single network with backpropagation, you evolve a whole population of them: each generation, the snakes that eat more apples and crash less get to pass on their genes, and over many generations the networks genuinely get good at the game.

What It Does

NEAT Algorithm Implementation

The complete algorithm, written from the paper: speciation, crossover and mutation, evolving not just the weights but the topology of the networks themselves.

Neural Network Architecture

My own neural network code — no ML libraries — with configurable input, hidden and output layers and ReLU/Tanh activations.

Fitness Evaluation

A fitness function that rewards eating apples, punishes crashing, and gives a small nudge for moving in the right direction.

Genome Serialization

The best genomes can be saved to disk and loaded later, either to keep training where you left off or just to show off the best snake.

Interactive Demonstration

You can watch the training live, speed it up or slow it down, and pick any generation to see how its best snake plays.

Problems I Ran Into

NEAT Algorithm Complexity

Algorithm Design
The problem

NEAT has a lot of moving parts — speciation, innovation numbers, crossover between networks with different topologies — and it's very easy to get one of them subtly wrong.

How I fixed it

I broke it into separate modules — genomes, speciation, crossover, mutation — and used innovation numbers to track each gene's history, which made crossing over different topologies much easier to reason about.

Performance Optimization

Computational Efficiency
The problem

Every generation means evaluating hundreds of networks playing full games of Snake. My early versions took ages to get through a single generation.

How I fixed it

I optimized the forward pass, tightened up the genome data structures, and added a step limit per evaluation plus configurable population sizes, so a bad snake can't waste everyone's time.

Fitness Function Design

Algorithm Tuning
The problem

Evolution is great at finding loopholes: reward the wrong thing and you get snakes that survive forever without ever going for an apple. Getting the incentives right took several tries.

How I fixed it

The final fitness function combines several signals: apples eaten (with growing bonuses for streaks), penalties for crashing, and a small reward for simply moving toward the apple to guide the early generations.

Genome Serialization

Data Persistence
The problem

Genomes don't have a fixed size — every network can have a different topology, with its own node and connection genes — so saving and loading them reliably wasn't trivial.

How I fixed it

I wrote a binary format that stores the network dimensions, weights, biases and metadata, with versioning and validation so genomes from different runs don't break each other.
