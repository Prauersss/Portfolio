Unity C# Marching Cubes Procedural Generation RPG Systems

Project Overview

A fully featured 3D mining RPG built from scratch in Unity, where players dig through a procedurally generated voxel world, collect minerals, upgrade tools, and interact with NPCs.

⛏️ Concept & Vision

Little Miner World is a 3D voxel mining RPG where the player explores and excavates a fully destructible terrain generated using the Marching Cubes algorithm. The game revolves around collecting minerals, managing an inventory, completing missions, and progressing through NPC-driven dialogue and a shop upgrade system.

Every system in the game was built from the ground up in Unity with C#, from the voxel terrain mesh generation to the UI widgets for the mineral codex, the in-game shop, and the cinematic triggers.

Key Systems & Features

🌍 Marching Cubes Terrain

Procedurally generated destructible 3D voxel terrain using a custom Marching Cubes implementation. The mesh updates in real time as the player mines.

⛏️ Pickaxe Mining System

Physics-based pickaxe tip detection that registers hits on the voxel terrain, triggers particle effects, and drops item pickups based on the mineral type struck.

🎒 Inventory & Pick-Up System

3D item pickup objects in the world that the player can collect into a unified inventory. Supports stackable materials and integrates with the shop and mission systems.

📖 Mineral Codex

An in-game encyclopedia that unlocks entries as the player discovers new minerals. Includes an animated unlock banner and per-entry widgets showing mineral details.

🛒 Tool Shop & Upgrades

A fully functional in-game shop where players spend collected materials to upgrade their pickaxe. Shop items display stats, requirements, and upgrade card animations.

💬 NPC Dialogue System

Conversation system with scriptable objects for dialogue, conversation effects, and conversation end triggers. Uses Text Animator for animated text display.

🎯 Mission System

Active mission panel that tracks and displays current objectives. Missions are driven by scriptable objects and update dynamically as the player progresses.

🎬 Cinematics & Save System

Cinematic trigger sequences for key story moments, and a persistent save system that stores player progress, inventory, and discovered minerals across sessions.

Technical Challenges & Solutions

🌍

Real-Time Voxel Mesh Updates

Procedural Generation
⚠ Challenge

Regenerating the Marching Cubes mesh every time the player mines caused visible frame hitches and made the interaction feel unresponsive.

✅ Solution

Implemented chunk-based terrain so only the affected chunk remeshes. Dirty flags track which chunks need updating, keeping unnecessary recalculations to zero.

⛏️

Accurate Pickaxe Hit Detection

Physics & Interaction
⚠ Challenge

Using standard collision detection on a dynamic mesh produced false positives and missed hits depending on swing angle and player speed.

✅ Solution

Created a dedicated PickaxeTip component that casts a short sphere overlap each frame during the swing animation window, identifying the closest voxel position with precision.

📖

Codex Unlock State Persistence

Data & Save System
⚠ Challenge

The Mineral Codex needed to remember which entries were unlocked across play sessions without tying the data to scene objects that might be destroyed.

✅ Solution

Utilized a ScriptableObject-based data model for the codex entries, with each entry's state (locked/unlocked) saved in the player prefs. Loaded and applied at runtime to update the UI.

🛒

Dynamic Shop Inventory

UI & Inventory Systems
⚠ Challenge

The shop inventory needed to update dynamically based on player progress and be independent of scene objects to avoid data loss on scene reloads.

✅ Solution

Implemented a data-driven shop system where the inventory is defined by scriptable objects. The shop UI reads from these objects directly, allowing for dynamic updates without scene dependency.
